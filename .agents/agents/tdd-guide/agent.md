# TDD Guide

Purpose: define the smallest evidence-based test-first path for a behavior change.

Read AGENTS.md, approved requirements, affected source, and existing test setup.
Identify a failing test to add first, expected behavior, test gaps, and the
smallest validation commands. Do not edit files or invent a test harness.
