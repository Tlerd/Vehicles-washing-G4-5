# Functional Requirements

| ID | Functional Requirement |
|---|---|
| FR-001 | The system shall allow the Parking Manager to manage parking facility information. |
| FR-002 | The system shall allow the Parking Manager to manage vehicle types. |
| FR-003 | The system shall allow the Parking Manager to configure floor or zone allocation by vehicle type. |
| FR-004 | The system shall allow the Parking Manager to manage parking slot statuses. |
| FR-005 | The system shall allow the Parking Manager to configure pricing and fee policies. |
| FR-006 | The system shall provide traffic volume reports by vehicle type. |
| FR-007 | The system shall provide revenue reports by vehicle type. |
| FR-008 | The system shall provide occupancy rate reports by vehicle type. |
| FR-009 | The system shall provide peak-hour reports by vehicle type. |
| FR-010 | The system shall allow the Parking Manager to track lost ticket cases. |
| FR-011 | The system shall allow the Parking Manager to flag license plate mismatches. |
| FR-012 | The system shall allow the Parking Manager to track overtime parking cases. |
| FR-013 | The system shall allow the Parking Manager to detect wrong-zone parking. |
| FR-014 | The system shall allow the Parking Manager to list unpaid vehicles. |
| FR-015 | The system shall allow Parking Staff to validate entry eligibility. |
| FR-016 | The system shall allow Parking Staff to capture or scan license plates on entry. |
| FR-017 | The system shall allow Parking Staff to guide vehicles to assigned floors or zones. |
| FR-018 | The system shall allow Parking Staff to create parking sessions with entry metadata. |
| FR-019 | The system shall allow Parking Staff to locate active sessions on exit. |
| FR-020 | The system shall allow Parking Staff to record exit times for sessions. |
| FR-021 | The system shall allow Parking Staff to calculate parking fees for sessions. |
| FR-022 | The system shall allow Parking Staff to collect parking fee payments. |
| FR-023 | The system shall allow Parking Staff to handle lost ticket exceptions. |
| FR-024 | The system shall allow Parking Staff to correct wrong vehicle information. |
| FR-025 | The system shall allow Parking Staff to handle overtime parking exceptions. |
| FR-026 | The system shall allow Parking Staff to handle wrong-zone parking exceptions. |
| FR-027 | The system shall allow Parking Staff to update slot status after exceptions. |
| FR-028 | The system shall allow Parking User/Driver to view operating hours. |
| FR-029 | The system shall allow Parking User/Driver to view supported vehicle types. |
| FR-030 | The system shall allow Parking User/Driver to view pricing and parking rules. |
| FR-031 | The system shall allow Parking User/Driver to view available slots by vehicle type. |
| FR-032 | The system shall allow Parking User/Driver to receive a parking ticket or code on entry. |
| FR-033 | The system shall allow Parking User/Driver to pay parking fees on exit. |
| FR-034 | The system shall allow Parking User/Driver to reserve a slot in advance when reservation is enabled. |
| FR-035 | The system shall allow Parking User/Driver to view current parking session details. |
| FR-036 | The system shall allow Parking User/Driver to pay for add-on services. |
| FR-037 | The system shall allow Parking User/Driver to submit a parking issue report. |
| FR-038 | The system shall allow the System Administrator to manage user accounts. |
| FR-039 | The system shall allow the System Administrator to manage roles. |
| FR-040 | The system shall allow the System Administrator to manage permissions. |
| FR-041 | The system shall allow the System Administrator to manage system configuration. |
| FR-042 | The system shall optimize slot allocation with AI to reduce search time and improve utilization. |
